How to run this U-SQL Sample Application

1. Submit scripts to (Local-machine) account
(Local-machine) account provides a shared local execution environment on your local machine. ADL Tools for VS has a built-in U-SQL local execution engine, and DataRoot folder points to a shared local storage account.
To set up data to run against (Local-machine) account, please copy the Samples folder from the TestData folder under project's directory to the local run DataRoot folder. The DataRoot folder is configured at:
  • Tools → DataLake → Options and Settings ... → Azure Data Lake → General → U-SQL Local Run → DataRoot
2. Submit scripts to (Local-project) account
(Local-project) account provides an isolated project level local execution environment for each run. It uses ADL Tools for VS's build in U-SQL local execution engine, and creates a temporary DataRoot folder under project working directory for each run.
No action is needed for setting up data to run against (Local-project) account. If you would like to change the input data source, please configure the folder path for input data through:
  • Right click project in Solution Explorer → Properties → Test Data Source
3. Submit scripts to ADLA account
ADLA account scales data process to multiple vertices on cloud. Every ADLA account has a default associated ADLS account as its storage.
To set up data to run against ADLA account, please upload the Samples folder from the TestData folder under project's directory to the root of the default ADLS account.
